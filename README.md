# Проект DocVQA Privacy Stage 1-2

Это учебный проект про обезличивание документов перед подачей в VQA пайплайн.
Идея простая. Берем пример из DocVQA, находим в OCR токенах ответ на вопрос, потом скрываем этот фрагмент в тексте и на изображении.

Сейчас в проекте нет обучения моделей и нет сравнения LayoutLMv3 и Donut.
Пока реализована только подготовка данных и базовый модуль обезличивания.

## Что сделано

1. Загрузка DocVQA из Hugging Face
2. Разбор OCR в удобный список токенов
3. Поиск позиции ответа в OCR
4. Получение bbox для найденного ответа
5. Маскирование области на картинке
6. Редактирование OCR токенов
7. Подсчет процента найденных ответов

## Подробно по функциям

### `src/load_data.py`

`DEFAULT_DATASET_NAME`
- Строка с именем датасета по умолчанию.
- Сейчас это `pixparse/docvqa-single-page-questions`.

`_to_pil_image(image_obj)`
- Внутренняя функция, приводит поле `image` к `PIL.Image`.
- Если уже `PIL.Image`, просто возвращает его.
- Если словарь с `bytes`, читает через `BytesIO`.
- Если словарь с `path`, читает файл по пути.
- Иначе пробует создать изображение через `Image.fromarray`.

`extract_ocr_tokens(example)`
- Забирает OCR из структуры `example["ocr_results"]["lines"][...]["words"]`.
- Для каждого слова берет:
  - текст `text`
  - координаты `bounding_box` или `bbox`
- Возвращает плоский список:
  - `{"text": ..., "bbox": ...}`
- Это основная функция, которую надо править первой, если поменяется формат датасета.

`load_docvqa(split="validation", limit=None)`
- Загружает split датасета через `datasets.load_dataset`.
- Если задан `limit`, берет первые `limit` примеров.
- Возвращает список обычных Python словарей, чтобы проще работать без привязки к объекту HF Dataset.

`show_example(example)`
- Печатает:
  - вопрос
  - ответы
  - первые OCR токены
- Потом показывает изображение через matplotlib.
- Удобно для быстрой ручной проверки, что OCR и bbox выглядят нормально.

### `src/anonymize.py`

`normalize_text(text)`
- Нормализация для сопоставления строк.
- Делает lowercase.
- Убирает пунктуацию через regex.
- Схлопывает лишние пробелы.
- Пример: `"$117,047"` превращается в `"117 047"`.

`_token_text(token)`
- Вспомогательная функция.
- Если токен это словарь, берет поле `text`.
- Иначе превращает токен в строку как есть.

`find_answer_span(ocr_tokens, answer)`
- Главная функция поиска ответа в OCR.
- Логика:
  1. Нормализует ответ и разбивает на части.
  2. Нормализует OCR токены.
  3. Разворачивает OCR в подпоследовательность под токенов и хранит маппинг к исходным индексам.
  4. Скользящим окном ищет точное совпадение.
  5. Возвращает `start_idx`, `end_idx`, `found`.
- Зачем разворачивать OCR:
  - чтобы ловить случаи вроде `3,700` и `$117 , 047`.

`_bbox_to_xyxy(bbox, image_size=None)`
- Приводит bbox к формату `(x1, y1, x2, y2)`.
- Поддерживает 3 вида bbox:
  - 4 числа
  - плоский полигон `x1,y1,x2,y2,...`
  - полигон из пар `[[x,y], ...]`
- Если координаты нормализованы от 0 до 1, масштабирует в пиксели по размеру изображения.

`span_bbox_from_tokens(token_entries, start_idx, end_idx)`
- Берет токены в диапазоне найденного ответа.
- Для каждого берет bbox и переводит в `xyxy`.
- Объединяет в один общий прямоугольник.
- Если bbox нет, возвращает `None`.

`evaluate_match_rate(dataset_subset, output_path=...)`
- Прогоняет выборку и считает, в скольких примерах найден ответ в OCR.
- Сейчас используется только первый вариант ответа `answers[0]`.
- Сохраняет статистику в JSON:
  - `total_checked`
  - `matched`
  - `match_rate`
  - первые примеры с деталями

`mask_image(image, bbox)`
- Делает копию изображения.
- Рисует черный прямоугольник по bbox.
- Возвращает замаскированную копию.

`redact_ocr_tokens(ocr_tokens, start_idx, end_idx, strategy="drop")`
- Обезличивает OCR текст в найденном диапазоне.
- `drop` удаляет токены из диапазона.
- `mask` заменяет текст на `[REDACTED]`.

## Ограничения текущей версии

1. Поиск ответа это exact match, без серьезного fuzzy.
2. В статистике пока берется только `answers[0]`.
3. Нет этапа инференса DocVQA моделей.
4. Нет финальных метрик EM и F1 по предсказаниям моделей.
