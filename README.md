# DocVQA - обезличивание документов (этапы 1-2)

Проект посвящен исследованию обезличивания документов в задаче Document VQA и анализу рисков восстановления скрытых полей.
В фокусе находятся два канала входных данных: изображение документа и OCR-текст.
На текущем этапе реализован базовый контур: локализация ответа в OCR, маскирование соответствующей области на изображении и редактирование OCR-токенов.

## Что умеет проект

- Загружает датасет DocVQA
- Находит ответ среди OCR-токенов документа
- Рисует прямоугольник поверх нужного места на изображении
- Стирает нужный фрагмент из OCR
- Считает, в скольких случаях ответ вообще удалось найти

## Основные функции

### `src/load_data.py`

- `load_docvqa(split, limit)` - загружает данные, `limit` ограничивает их количество
- `extract_ocr_tokens(example)` - вытаскивает все OCR-токены с координатами
- `show_example(example)` - печатает вопрос, ответы и показывает картинку

### `src/anonymize.py`

- `normalize_text(text)` - переводит текст в нижний регистр, убирает знаки препинания и лишние пробелы. Нужно потому что OCR и ответы часто написаны немного по-разному (`$117,047` и `117 047` - это одно и то же)
- `find_answer_span(ocr_tokens, answer)` - ищет ответ в OCR. Сначала нормализует оба текста, потом идёт скользящим окном по токенам. Если нашёл - возвращает индексы начала и конца
- `span_bbox_from_tokens(...)` - по найденным токенам строит прямоугольник для маски
- `mask_image(image, bbox)` - закрашивает нужную область на картинке
- `redact_ocr_tokens(...)` - удаляет нужные токены из OCR
- `evaluate_match_rate(...)` - считает процент документов, где ответ удалось найти, и сохраняет в `outputs/stats/answer_match_rate.json`

## Что показывают ноутбуки

**`01_inspect_docvqa.ipynb`** - загружаем датасет и смотрим как он устроен.

**`02_anonymization_demo.ipynb`** - считаем match rate, смотрим примеры до и после маскирования, разбираем случаи где ответ не нашёлся.
